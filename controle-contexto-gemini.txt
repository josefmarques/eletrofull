    Contexto do Projeto EletroFull (Checkpoint 01)

    1. Ambiente e Stack:

        SO: Pop!_OS (Linux) local.

        Infra: Docker Compose com containers -local (backend-local, frontend-local, postgres-local, nginx-proxy-local).

        Backend: Python 3.11, FastAPI, SQLModel (SQLAlchemy + Pydantic), Alembic para migrações.

        Frontend: Next.js 14+ (App Router), TypeScript, TailwindCSS 4, shadcn/ui.

        Banco: PostgreSQL 17 (Bancos: eletrosil_db e emarques_db).

    2. Estado Atual do Sistema:

        Multi-Tenancy: Funcionando via subdomínios locais (eletrosil.localhost, emarques.localhost).

        Roles (RBAC): admin (Global), manager (Gerente), operator (Caixa) e vendedor (Vendedor de Balcão).

        Módulo de Vendas: Vendas vinculadas a seller_id e customer_id com cálculo automático de commission_value.

        Módulo de Orçamentos: Implementado em /quotes. Permite salvar propostas e convertê-las em vendas sem baixar estoque prematuramente.

        Clientes: Tabela centralizada por rede (Tenant) com histórico de unidade de origem.

        Padronização: URLs em inglês (/quotes, /branches, /audit, /history), mas interface 100% em português brasileiro.

    3. Dados e Migração:

        Base original vinda de Firebird 5.0 (migração parcial via scripts Python).
